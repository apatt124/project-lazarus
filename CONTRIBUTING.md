# Contributing to Project Lazarus

Thank you for your interest in contributing to Project Lazarus.

## Important Note

This is a personal medical history management system. Contributions should focus on:
- Bug fixes
- Security improvements
- Documentation enhancements
- Feature suggestions

## Getting Started

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Submit a pull request

## Code Standards

- Follow PEP 8 for Python code
- Use ESLint/Prettier for JavaScript/TypeScript
- Include docstrings and comments
- Write unit tests for new features

## Security

- Never commit credentials or API keys
- Report security vulnerabilities privately
- Follow HIPAA compliance guidelines
- Use encryption for all PHI

## Testing

- Test with sample data only (no real PHI)
- Include unit tests
- Verify AWS resource tagging
- Check cost implications

## Documentation

- Update README for new features
- Document API changes
- Include setup instructions
- Add inline code comments

## Pull Request Process

1. Update documentation
2. Add tests
3. Ensure CI passes
4. Request review
5. Address feedback

## Questions?

Open an issue for discussion before major changes.
